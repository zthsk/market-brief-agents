# Market Brief Agents

Market Brief Agents is a local-first AI engineering project for turning market events into
evidence-grounded short-form financial briefings. It combines data collection, source review,
script generation, retrieval, agent orchestration, media rendering, and quality gates.

It is built as a portfolio-grade system design project, not as a trading platform. The generated
content is educational and must not be treated as buy, sell, or hold advice.

## Why This Project Exists

Modern finance content pipelines need more than a prompt. They need source selection, traceable
state, structured generation, output validation, artifacts for review, and a way to resume or
inspect long-running workflows. This repo demonstrates those AI engineering concerns in a compact,
local-first application.

## System Overview

```text
Market data, filings, news, synthetic demo fixtures
        |
        v
Event detection and candidate ranking
        |
        v
Research source collection, source tiering, approval gate
        |
        v
RAG retrieval over approved evidence
        |
        v
LangGraph-compatible agent workflow
        |
        v
Typed script manifest -> validated script package
        |
        v
Audio/assets/video rendering -> quality reports -> review dashboard
```

Key engineering features:

- Stateful agent pipeline with a LangGraph-compatible graph and local fallback runner.
- Synthetic no-key demo path for public portfolio review.
- Source-tier policy for primary, reputable, commentary, and discovery-only evidence.
- Pydantic validation for generated scripts, scene plans, templates, and production metadata.
- Deterministic local fallbacks when LLM provider keys are not configured.
- Audit bundles for prompts, raw model responses, validation errors, scripts, render manifests, and quality reports.
- Streamlit operations dashboard, Next.js public site shell, and Remotion video renderer.

## Quick Start

Prerequisites:

- Python 3.11+
- `uv`
- Node.js/npm for the frontend and Remotion renderer
- FFmpeg for video rendering

```bash
uv sync --extra dev
npm install --prefix frontend
npm install --prefix remotion
cp .env.example .env
uv run marketbrief init-db
```

Run the public synthetic demo without API keys:

```bash
uv run marketbrief run-agent-pipeline --demo --thread-id demo-001 --skip-render
uv run marketbrief inspect-agent-run --thread-id demo-001
```

Pause before generation and resume from the same thread:

```bash
uv run marketbrief run-agent-pipeline --demo --thread-id review-001 --skip-render --interrupt-before-script
uv run marketbrief inspect-agent-run --thread-id review-001
uv run marketbrief run-agent-pipeline --thread-id review-001 --resume --skip-render
```

This creates ignored local runtime artifacts under `data/` and `outputs/`. Nothing generated by the
demo is intended to be committed.

## Optional Provider Configuration

Real data collection and AI generation are optional. Configure only the providers you want to use:

```env
SEC_USER_AGENT="Market Brief Agents local demo contact@example.com"
AI_PROVIDER="gemini"
TTS_PROVIDER="gemini"
GEMINI_API_KEY=""
GEMINI_TEXT_MODEL="gemini-2.5-flash"
GEMINI_TTS_MODEL="gemini-2.5-flash-preview-tts"
GEMINI_TTS_VOICE="Kore"
WEB_SEARCH_PROVIDER="exa"
EXA_API_KEY=""
MARKET_BRIEF_DB_PATH="data/market_brief_agents.db"
MARKET_BRIEF_LOG_LEVEL="INFO"
```

Without provider keys, the synthetic demo and deterministic local script generation still work.

## Repository Map

- `agentic/`: LangGraph-compatible orchestration and RAG retrieval.
- `backend/`: CLI and Streamlit review dashboard.
- `jobs/`: Batch and scheduled pipeline workflows.
- `services/`: Data collection, source review, LLM/TTS wrappers, script generation, demo fixtures.
- `media_engine/`: Story schemas, templates, production plans, renderer adapters, quality gates.
- `remotion/`: TypeScript Remotion video composition.
- `frontend/`: Public Next.js site shell.
- `examples/synthetic/`: Public demo fixture notes.
- `docs/`: Architecture and publication docs.
- `tests/`: Unit tests for collectors, pipeline logic, media engine, research, scripts, and agents.

## Public Safety

The repo is designed so generated artifacts stay local:

- `.env`, databases, logs, videos, audio, rendered bundles, and provider responses are ignored.
- Public demo data is synthetic.
- The public-clean scanner rejects common secret patterns and generated artifact extensions.
- The script generator validates source ids, blocks financial-advice language, and writes audit artifacts.
